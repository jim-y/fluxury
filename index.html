Welcome to Fluxury's project-page.

This is a short share-how about Fluxury's implementation details. I plan to keep it clear and simple.

Build tool

I switched from Grunt to npm because I wanted to get rid of the numerous grunt-* dependencies. Using npm as a build tool in such a small project (as well as in biggers) is easy, and you don't need any dependency doing it. For more about this topic read this blog post: How to use npm as a build tool

Used technologies

  • browserify: for bundling up CommonJS modules for the browser
  • react: for achieving component driven programming style
  • flux: for managing data flow through the application. ~Observer pattern, also known as Publish-Subscribe
  • immutable: to hold the application state in immutable data sets
  • react-router: I found this library really handy for inter-screen transitions

Immutability generally makes your state management side effect free by creating new data sets on every mutations. In the skeleton app I've just made some pin-points on how to use the library. React-router makes the transitions between your pages/screens a charm. Ever made your mind around how to switch pages in an SPA? This library can be an answer to your questions.

In short, you make a list about your routes/pages, like [home, contact, shop, projects, etc...], and you define handler React-components for these routes, and you will put a placeholder (Director) somewhere in your masterpage. Then, whenever you want redirect the user from page A to page B, you only need to create a Link component which tells the router where you want to navigate. (e.g Home). Then, the placehold will hold a new value, and that will be your component defined for the Home route.

Devdependencies

  • jsxhint: for linting your code.
  • node-jscs: jscs needs to be pre-installed globally, but the app holds a build target, and an airbnb preset for coding-style validation. Sadly jscs doesn't support ES6 well, so we get some strange errors for our ES6 classes in the store(s).
  • reactify: browserify-transform for ES6-to-ES5 compilation.
  • uglify: minifying the codebase. TBD
  • watchify: for live-rebuild. TBD

Details

I used ES6 classes in the Store, and ES6 destructuring among the app.

Destructuring:

{ Home, Contact, PirateBadge } = require('./components/Components')

This is the same, as writing:

Home = require('./components/Components').Home,
Contact = require('./components/Components').Contact,
PirateBadge = require('./components/Components').PirateBadge
